# L2 个人助手规范

## 概述

个人助手（Personal Assistant）是人类员工的数字化镜像。它习得主人的知识、经验、权限和操作能力，能以假乱真地执行工作任务。当 L1 运营智能体的流程中遇到「人节点」时，直接调度对应的 L2 个人助手接管执行。

## 认知镜像管线

```
原始资料                             个人助手知识库
┌──────┐    ┌──────────┐    ┌──────────────────┐
│ 文档  │───→│ 知识抽取   │───→│ 结构化知识图谱    │
├──────┤    │ Agent    │    ├──────────────────┤
│ 邮件  │───→│          │───→│ 经验规则库        │
├──────┤    │          │    ├──────────────────┤
│ 聊天  │───→│          │───→│ 决策案例库        │
├──────┤    └──────────┘    ├──────────────────┤
│ 操作  │───→ 工具使用习得 ──→│ 操作手册（Skill） │
└──────┘                    └──────────────────┘
```

### 习得内容

| 维度 | 习得来源 | 输出 |
|------|---------|------|
| **事实知识** | 文档、邮件、知识库 | 结构化知识图谱 |
| **经验判断** | 历史决策记录 | 经验规则库 |
| **操作技能** | 系统操作日志 | Tool/Skill 注册表 |
| **沟通风格** | 聊天记录、邮件 | 风格模型 |

## 典型角色

### 🧮 财务行政助手

- **职责**: 报销审核、发票核验、预算跟踪、付款排期
- **接入系统**: 费控系统、ERP 财务模块、银行接口
- **习得**: 财务审核经验、费用标准知识、报销政策理解

### 💻 产品研发助手

- **职责**: 技术文档编写、代码 Review、需求分析、技术方案
- **接入系统**: GitLab/GitHub、JIRA、Confluence
- **习得**: 团队编码规范、技术栈知识、架构决策记录

### 📊 售前方案助手

- **职责**: 方案编写、竞品分析、客户沟通、标书准备
- **接入系统**: CRM、标书系统、知识库
- **习得**: 方案模板、客户行业知识、竞品信息

### 📅 秘书

- **职责**: 日程管理、会议安排、任务委派、出差协调
- **接入系统**: 日历、OA 系统、通讯工具
- **习得**: 老板的时间偏好、优先级判断、沟通习惯
