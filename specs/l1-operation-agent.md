# L1 运营智能体规范

## 概述

运营智能体（Operation Agent）按企业价值流和资源域拆分，是三层架构中的执行中坚。每个运营智能体拥有独立的事件感知、认知推理和执行能力，形成领域级业务闭环。

## 八大价值流

### 📈 商机捕获智能体

| 维度 | 说明 |
|------|------|
| **事件** | 招标公告、市场需求信号、客户咨询 |
| **流程** | 线索采集 → AI 评分 → 跟进分配 → 转化追踪 |
| **规则** | 评分规则（金额/匹配度/紧迫度）、分配规则 |
| **行为** | 分配商机、发起跟进、更新 CRM |

### 📋 销售订单智能体

| 维度 | 说明 |
|------|------|
| **事件** | 客户下单、价格变更、订单修改 |
| **流程** | 订单录入 → 库存校验 → 价格审批 → 发货安排 |
| **规则** | 价格折扣权限、信用额度校验 |
| **行为** | 锁定库存、发出发货指令、更新订单状态 |

### 📄 合同审核智能体

| 维度 | 说明 |
|------|------|
| **事件** | 合同起草完成、合同到期、条款变更 |
| **流程** | 草拟 → AI 审查 → 合规校验 → 签署 → 归档 |
| **规则** | 46号令合规规则、金额审批阈值、黑名单校验 |
| **行为** | 标记风险条款、发起电子签章、归档合同 |

### 🔬 研发交付智能体

| 维度 | 说明 |
|------|------|
| **事件** | 需求变更、发布计划、Bug 上报 |
| **流程** | 需求分析 → 任务分配 → 开发 → 测试 → 发布 |
| **规则** | 优先级排序、资源分配策略 |
| **行为** | 创建任务、分配资源、自动化测试触发 |

### 📦 采购管理智能体

| 维度 | 说明 |
|------|------|
| **事件** | 采购申请、供应商报价、入库通知 |
| **流程** | 需求确认 → 供应商寻源 → 比价议价 → 下单 → 验收 |
| **规则** | 供应商准入规则、比价规则、框采优先 |
| **行为** | 发出 RFQ、创建采购订单、触发付款 |

### 💰 财务结算智能体

| 维度 | 说明 |
|------|------|
| **事件** | 发票开具、付款到期、对账差异 |
| **流程** | 发票核验 → 三单匹配 → 付款审批 → 付款执行 |
| **规则** | 付款周期规则、预算控制规则、税务合规 |
| **行为** | 核验发票、发起付款、生成凭证、上报监管 |

### 👥 人力资源智能体

| 维度 | 说明 |
|------|------|
| **事件** | 入职申请、离职申请、绩效考核启动 |
| **流程** | 招聘 → 面试 → 入职 → 培训 → 考核 → 离职 |
| **规则** | 编制约束、薪酬标准、晋升规则 |
| **行为** | 发起招聘流程、录用通知、更新花名册 |

### 🎧 客户服务智能体

| 维度 | 说明 |
|------|------|
| **事件** | 客户投诉、工单创建、服务满意度评分 |
| **流程** | 工单接收 → 智能分配 → 处理 → 回访 → 关闭 |
| **规则** | 分级响应（L1~L4）、SLA 规则、升级规则 |
| **行为** | 分配工单、发送通知、生成处理报告 |

## 技术架构

```
┌─────────────────────────────────────────────┐
│  运营智能体 (L1)                              │
│                                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ 领域知识库 │  │ 规则引擎  │  │ 流程引擎  │  │
│  │ (MD文件)  │  │ (E5M)    │  │(Temporal) │  │
│  └──────────┘  └──────────┘  └──────────┘  │
│       │              │              │        │
│  ┌────▼──────────────▼──────────────▼─────┐  │
│  │        事件感知 → 认知 → 行动闭环       │  │
│  └───────────────────────────────────────┘  │
│                                              │
│  遇「人节点」 → 调度 L2 个人助手              │
└─────────────────────────────────────────────┘
```

## 协同模式

- **上下游协作**: 采购智能体 → 财务智能体（付款）
- **L0 编排**: L0 决策智能体安排跨域多 L1 协作
- **L2 调度**: 遇到需要人类参与的节点，调度对应 L2 个人助手执行
